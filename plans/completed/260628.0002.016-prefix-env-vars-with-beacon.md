# Task Breakdown

## Overview

Namespace all operator-supplied environment variables with a `BEACON_` prefix to prevent
collision with other projects sharing the same server environment. Six variables are renamed;
all usage sites already go through Go `const` literals, so only the constant values change —
all 100+ call sites follow automatically.

## Assumptions

- The Go const identifiers (`envDsnSqliteDB`, `envDsnTelegramBOT`, etc.) are NOT renamed —
  only their string values change.
- The `.env` / `.env.example` files are permission-guarded and must not be edited; the
  operator will apply the hand-edit list in this plan.
- `.github/workflows/` does not reference the six variable names directly (confirmed by grep).
- Archival records (`plans/completed/`, `plans/history/`, `docs/reviews/`, `plans/HANDOFF-*.md`,
  `plans/007-*.md`, `plans/012-*.md`, `plans/014-*.md`) are left untouched.
- Test-only synthetic env names (`PROXY_URL_TEST_ABSENT_XYZ`, `PROXY_URL_TEST_VALID`) are left
  verbatim; only the test assertion that checks the emitted log line prefix is updated to match
  the new `BEACON_PROXY_URL=` log output.

## Tasks

### Task 1: Update const values in `cmd/*` entry points

- Description: Change the string VALUE of each env-var const in the five `cmd/*` entry points.
  Keep identifier names (`envDsnSqliteDB`, etc.) unchanged.
- Files: `cmd/migrator/main.go`, `cmd/notifier/main.go`, `cmd/web/main.go`,
  `cmd/collector/main.go`, `cmd/doctor/rulegen.go`
- Acceptance Criteria: `grep -r '"SQLITEDB_DSN"\|"TELEGRAMBOT_DSN"\|"AI_PRIMARY_DSN"\|"AI_FALLBACK_DSN"\|"CHROMIUM_PATH"\|"PROXY_URL"' cmd/` returns zero hits.
- Pitfalls & edge cases: `cmd/web/main.go` also has a runtime `log.Fatalf` string that names
  `TELEGRAMBOT_DSN` — update that too.
- Complexity: Easy

### Task 2: Update doc-comments and log line in `internal/`

- Description: Update package/function doc-comments and the one runtime log.Printf label that
  exposes the var name in proxyutil.go. Also update the test assertion that verifies the log
  prefix.
- Files: `internal/tools/proxyutil/proxyutil.go`, `internal/tools/proxyutil/proxyutil_test.go`,
  `internal/tools/rateextractor/rateextractor.go`,
  `internal/application/rulegen/chromedpfetcher_test.go`,
  `internal/infrastructure/telegrambot/tbotclient.go`
- Acceptance Criteria: No old var names appear in any of these files (except synthetic test env
  names `PROXY_URL_TEST_*`).
- Pitfalls & edge cases: `proxyutil_test.go` assertion `assert.Contains(t, logLine, "PROXY_URL="`
  still passes even without a fix (substring match), but it would check the wrong string — update
  it to `"BEACON_PROXY_URL="` for correctness.
- Complexity: Easy

### Task 3: Update usage help text in `cmd/doctor/rulegen.go`

- Description: The `printRulegenUsage` function and package godoc block contain literal var names
  as help-text strings. Update all six to the prefixed form.
- File: `cmd/doctor/rulegen.go`
- Acceptance Criteria: Running `./build/doctor rulegen --help` displays `BEACON_*` names only.
- Pitfalls & edge cases: The godoc block at the top of the file (func godoc via `go doc`) also
  lists the var names — update both the godoc and the printed usage.
- Complexity: Easy

### Task 4: Update operator docs

- Description: Rename all occurrences of the six vars in `README.md`, `deploy/README.md`,
  `cmd/doctor/README.md`, and `CLAUDE.md`.
- Acceptance Criteria: `grep -r 'SQLITEDB_DSN\|TELEGRAMBOT_DSN\|AI_PRIMARY_DSN\|AI_FALLBACK_DSN\|CHROMIUM_PATH\|PROXY_URL' README.md deploy/README.md cmd/doctor/README.md CLAUDE.md` returns zero hits.
- Pitfalls & edge cases: `deploy/README.md` has a shell snippet with an awk command that uses
  `SQLITEDB_DSN` as a literal pattern. Update the awk pattern as well.
- Complexity: Easy

### Task 5: Verify build and tests

- Description: Run `CGO_ENABLED=0 make build` and `make test` (race mode). Both must pass.
- Acceptance Criteria: No compilation errors, all tests pass.
- Complexity: Easy

## Execution Order

1 → 2 → 3 (3 is partly covered by 1) → 4 → 5

## Risks

- **Deploy-time coordination**: the server EnvironmentFile (`/opt/beacon/.env`) must be updated
  to the new names BEFORE the next deploy. If the old names are in `.env` and the new binary
  starts, `dsninjector.Unmarshal` will not find the key and the binary will `log.Fatalf` on
  startup. This is not data-loss, but it is an outage until the file is patched.
- **`.env.example`**: if `.env.example` exists and is shared, the operator must update it too
  (see hand-edit list below).

## Trade-offs

- Identifier names (`envDsnSqliteDB` etc.) are left unchanged because renaming them provides no
  operator-visible benefit and would touch every call site unnecessarily.
- Archival plan files are not rewritten — falsifying historical records is worse than
  leaving stale names in closed documents.

## `.env` Hand-edit List

The operator must apply these substitutions to `/opt/beacon/.env` (and `.env.example` if it
exists) BEFORE the next deploy:

```
# Old → New
SQLITEDB_DSN=...     →  BEACON_SQLITEDB_DSN=...
TELEGRAMBOT_DSN=...  →  BEACON_TELEGRAMBOT_DSN=...
PROXY_URL=...        →  BEACON_PROXY_URL=...
AI_PRIMARY_DSN=...   →  BEACON_AI_PRIMARY_DSN=...
AI_FALLBACK_DSN=...  →  BEACON_AI_FALLBACK_DSN=...
CHROMIUM_PATH=...    →  BEACON_CHROMIUM_PATH=...
```

Example (for the three most common vars):
```
BEACON_SQLITEDB_DSN=sqlite://_:_@_:_//opt/beacon/beacon.sqlite
BEACON_TELEGRAMBOT_DSN=tbot://<admin_chat_id>:@<bot_token>/
BEACON_PROXY_URL=http://127.0.0.1:7788
```
