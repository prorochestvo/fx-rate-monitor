### 2.10 `571ee96` — perf(rateextractor,collection): share one chromedp allocator per tick

**Что:** Bundle 8 — Task 13. Один shared Chromium subprocess per
collector tick вместо cold-start per source.

- `internal/tools/rateextractor/chromedp.go`:
  - Новый `RunBatch(ctx, []*RateSource) map[string]error`.
  - Один `ExecAllocator` через `slices.Clone(DefaultExecAllocatorOptions)`
    + `Headless / DisableGPU / NoSandbox / disable-blink-features` +
    `ExecPath` (если chromiumPath не пуст).
  - Sources обрабатываются sequentially (chromedp browser contexts от
    одного allocator не concurrency-safe).
  - 30s timeout scoped per source.
  - `defer cancelAlloc()` reaps subprocess на любом пути.
  - Между sources — `ctx.Err()` check; на cancellation теги остатки
    cancellation-error вместо silent drop.
  - Старый `Run(ctx, *RateSource)` теперь one-element wrapper над
    `RunBatch`.
  - "Trust boundary" godoc на `RunBatch` — `--no-sandbox` + shared
    `--user-data-dir` означают shared network service / DNS cache /
    HTTP auth credential cache. Callers должны batch'ить только
    operator-vetted https URLs без credentials.
  - `validateNavigableURL` теперь rejects userinfo (`u.User != nil`)
    — error message НЕ цитирует URL чтобы credentials не попали в log.
- `internal/application/collection/rateagent.go`:
  - Новый `chromedpBatchExtractor` interface (только `RunBatch`).
  - `RateAgent.execution` partitions sources by `FetcherKind`: plain
    sources идут per-source, chromedp — одним `RunBatch`.
  - Execution_history persistence runs в final loop в
    source-declaration order под `context.Background()` чтобы SIGTERM
    не дропал rows для уже завершённых sources.
- `internal/tools/rateextractor/chromedp_test.go` — `TestChromedpRateExtractor_RunBatch`:
  empty batch no-op, pre-cancelled ctx, tombstone short-circuit.
- `internal/application/collection/rateagent_test.go` — mock получает
  `RunBatch` method.
- `deploy/README.md` — paragraph про ~30s × N budget для chromedp ticks.

**Зачем:** Каждый chromedp source стартовал свой Chromium subprocess.
N sources = N cold-starts. С batch'ем — один cold-start total.

**Как проверять:**
- Запустить collector с >1 chromedp source — должен видеть один
  Chromium процесс, sequential per-source navigation.
- `defer cancelAlloc()` reaps subprocess — `ps aux | grep chromium`
  после tick'а должно быть пусто.
- Trade-off: chromedp execution_history rows теперь сохраняются после
  всего batch'а; `persistCtx := context.Background()` обеспечивает
  durability на SIGTERM. Документировано.

**Риски:** Средние. Shared Chromium расширяет blast radius для
malicious source (документировано в trust boundary). Seed set
operator-vetted, so OK.
